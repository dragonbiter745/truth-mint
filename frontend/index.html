How The Truth is Forged

Decentralized verification is a three-stage cryptographic process.

01

Request: The AI Query

An AI agent asks for external data (e.g., "What is the BTC price?" or "Did event X occur?"). The request is routed to the decentralized Flare Oracle network.

[AI Agent] → [Truth API]
02

Verify: FTSO & Attestation

The Flare Time Series Oracle (FTSO) or Flare Data Connector (FDC) sources, aggregates, and cryptographically signs the data based on consensus from 100+ decentralized providers.

[Oracle] + [FTSO/FDC] → [Consensus]
03

Prove: Anchor & Return

The verified data is anchored to a truth contract on the Flare blockchain. A proof object (signature + timestamp) is generated and returned to the querying AI agent.

[Proof] → [Agent Response]
