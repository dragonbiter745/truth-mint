API Documentation

The programmatic interface to cryptographically verifiable data for AI Agents.

Introduction

The Truth API is a RESTful service designed for seamless integration with large language models and autonomous agents. Every response includes a canonical proof object anchored on the Flare blockchain.

CURL 
curl -X GET 'https://api.truth.flare/price/ftso?asset=BTC/USD' \
  -H 'X-API-KEY: your_api_key'

All endpoints require a valid **X-API-KEY** passed in the header. API keys can be managed in the Agent Cockpit.

Authentication

Authentication is handled via a single, long-lived API key (X-API-KEY) managed by the user's Smart Contract wallet on Flare. Keys are non-expiring but can be manually revoked or rotated via the Dashboard.

JAVASCRIPT 
const apiKey = 'sk_live_...'; // Retrieved from Dashboard
const url = 'https://api.truth.flare/price/ftso?asset=ETH/USD';

fetch(url, {
    method: 'GET',
    headers: {
        'X-API-KEY': apiKey,
        'Content-Type': 'application/json'
    }
}).then(res => res.json()).then(data => console.log(data));

GET /price/ftso

Retrieves the latest verified price feed (within the last 90 seconds) using the Flare Time Series Oracle (FTSO). This is the fastest query type.

Parameters

asset String (Required) The asset pair to query (e.g., BTC/USD, ETH/USDT).

Example Response (Success)

JSON 
{
    "status": "success",
    "data": {
        "value": "3561.88",
        "asset": "ETH/USD",
        "proof": {
            "signature": "0x8a92e10c...",
            "txHash": "0x12c4d678...",
            "timestamp": 1733367900
        }
    }
}

POST /attest/event

Initiates a request to the Flare Data Connector (FDC) to attest to a specific event or data hash. This endpoint triggers an on-chain anchoring process and is subject to network fees.

Request Body

dataHash String (Required) The unique SHA-256 hash of the data or event to be attested. eventID String (Optional) A custom ID for tracking complex attestations (Legal Contract ID, Financial Period, etc.).

Example Request (CURL)

CURL 
curl -X POST 'https://api.truth.flare/attest/event' \
  -H 'X-API-KEY: your_api_key' \
  -H 'Content-Type: application/json' \
  -d '{
    "dataHash": "0x4ae891c3d1f0d...",
    "eventID": "LegalContract-1024"
  }'

GET /proof/{txHash}

Retrieves the full canonical proof object, including Oracle Signatures, associated with a specific Flare transaction hash (TX Anchor).

Canonical Proof Object Schema

The standard structure for verifiable truth returned by the Truth API. This object contains the cryptographic evidence necessary for local signature verification.

Schema Definition

JSON 
{
    "proofId": "uint256 (BigNumber)",
    "dataValue": "string",
    "timestamp": "uint64 (Epoch)",
    "txHash": "string (0x...)",
    "oracleSource": "string (FTSO/FDC)",
    "signature": "string (E-25519 or other elliptic curve sig)"
}

Error Codes

Common HTTP error codes returned by the API.

401 Unauthorized Invalid or missing X-API-KEY. Check dashboard for active key. 429 Rate Limit Too many requests in a short time. Slow down automated agent queries. 504 Gateway Timeout Backend failed to receive proof confirmation from the blockchain within the timeout (possible high gas fees or congestion).
